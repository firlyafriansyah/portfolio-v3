Konflik di Repositori Github

Dalam sebuah kolaborasi timbul suatu masalah bukanlah hal yang aneh, melainkan sudah menjadi suatu hal wajar. Begitu pula saat para developer sedang melakukan kolaborasi pada sebuah projek sering mendapat sebuah permasalahan atau konflik yang terjadi. Dalam kolaborasi pembuatan projek developer akan mendapatkan tugasnya masing - masing, namun terkadang akan ada suatu saat dimana beberapa developer mengerjakan suatu kasus pada projek di file yang sama dengan developer lainnya.

Ketika dua orang developer mengerjakan kasus pada file yang sama dan seorang developer sudah melakukan push perubahan ke repositori server yang mereka gunakan, maka ketika developer yang satu lagi akan melakukan push juga ini akan menimbulkan konflik, karena file yang digunakan oleh developer kedua ini sudah tidak sama dengan aslinya dikarenakan developer pertama sudah melakuakan perubahan terhadap file tersebut, ini akan mengakibatkan developer kedua tidak bisa menambahkan perubahan yang yang terjadi file tersebut.

Disinilah sebuah Branch dibutuhkan, Branch dapat diistilahkan sebuah folder dalam repositori, namun folder ini bukanlah folder utama dari repositori ini jadi apapun yang terjadi pada folder ini tidak akan berpengaruh pada file atau folder yang utama. dan biasanya setiap developer yang tergabung di projek ini akan memiliki branch masing agar konflik yang serupa dapat dihindarkan.

Lalu bagaimana agar perubahan yang mereka lakukan dapat di satukan?

Dalam git dikenal dengan istilah Merge, merge ini merupakan cara yang untuk menggabungkan satu branch dengan branch lain. Setelah dilakukan compare terhadap perubahan yang dilakukan para developer maka salah satu anggota tim akan melakukan marge pada branch mereka ke branch ini atau sering disebut master/main branch.

Hal ini akan mempermudah dan menghindari konflik pada setiap perubahan yang dilakukan oleh para developer.
